  Funzionalita' implementate

  Menu SiFirma con 3 voci:
  1. Pannello Amministrazione - Configura le credenziali e le impostazioni
  2. Avvia Firma Documento - Lancia il processo di firma
  3. Verifica Stato Processo - Controlla lo stato tramite UID

  Pannello Amministrazione (ConfigureAdmin):
  - URL API SiFirma (con default al test environment)
  - API Key, API Secret
  - Partita IVA / Codice Fiscale
  - Tipo firma default (FES/FEA/FEQ)
  - Tipo posizionamento firma (Coordinate / Acrofield / MarcatoriV2 per i tag
  nel documento)
  - Giorni scadenza default
  - Invia email ai firmatari
  - URL redirect post-firma e WebHook callback
  - Pulsante "Testa Connessione" → verifica via GET /api/v2/info/versioni

  Dialog Avvia Firma (StartSigning):
  - Nome processo e nome file PDF
  - Data scadenza (pre-calcolata dai giorni configurati)
  - Tipo firma (modificabile per ogni processo)
  - Dati Firmatario 1 (obbligatorio): Nome, Cognome, CF, Email, Telefono
  - Dati Firmatario 2 (opzionale): stessi campi

  Controlli e validazioni:
  - Verifica configurazione completa prima di qualsiasi operazione
  - Verifica documento aperto e tipo supportato (Writer/Calc/Impress)
  - Validazione data scadenza (formato e futura)
  - Validazione firmatari (nome+cognome + CF o email obbligatori)
  - Gestione errori HTTP con estrazione del corpo della risposta
  - File config salvato con permessi 600 (solo utente proprietario)

  API SiFirma utilizzate (WebAPI V2)

● Causa del problema: i dialog UNO creati programmaticamente in Python
  richiedono che venga chiamato createPeer(toolkit, None) prima di execute().
  Senza questo passaggio, il dialog viene costruito in memoria ma non viene mai
  visualizzato a schermo.

  # Rimuovi la versione precedente
  unopkg remove com.sifirma.sign

  # Installa la nuova
  unopkg add

  estensioni: rimuovi "SiFirma Sign" e aggiungi il nuovo .oxt.
